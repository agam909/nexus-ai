# Production Deployment Guide

Follow these steps **in order**. Total time: ~30 minutes. Total cost: **₹0**.

You will collect 5 secrets and paste them into Render. Keep a notepad open.

---

## 1. Groq (LLM) — ~2 min

1. Visit <https://console.groq.com/keys>
2. Sign in with Google.
3. Click **Create API Key**. Name it `nexus`.
4. Copy the key starting with `gsk_…`
5. ☑ Save as **`GROQ_API_KEY`**

---

## 2. HuggingFace (Embeddings) — ~3 min

1. Visit <https://huggingface.co/join> and sign up (just email).
2. Verify your email.
3. Go to <https://huggingface.co/settings/tokens>
4. Click **Create new token** → **Read** scope → name it `nexus`.
5. Copy the token starting with `hf_…`
6. ☑ Save as **`HF_API_TOKEN`**

---

## 3. Qdrant Cloud (Vector DB) — ~5 min

1. Visit <https://cloud.qdrant.io/> → sign in with Google.
2. Click **Clusters → + Create**.
3. **Free tier**, region: closest to you (e.g. `Singapore` or `Mumbai` if available). Cluster name: `nexus`.
4. Wait ~1 min for the cluster to become **Healthy**.
5. Click the cluster → **API keys → Create API key** → copy the key.
6. From the cluster overview, copy the **Endpoint URL** (looks like `https://abcd-efgh.us-east.aws.cloud.qdrant.io:6333`).
7. ☑ Save as **`QDRANT_URL`** and **`QDRANT_API_KEY`**

---

## 4. Neon (Postgres Database) — ~3 min

1. Visit <https://neon.tech/> → sign in with GitHub or Google.
2. **Create project**: name `nexus`, region closest to Render (`Oregon US West` recommended to pair with the Render region we'll pick).
3. On the project dashboard, copy the **Connection string** (it says `postgres://user:pass@host/db?sslmode=require`).
4. ☑ Save as **`DATABASE_URL`**

---

## 5. Render (Hosting) — ~10 min

1. Visit <https://render.com/> → sign in with GitHub.

2. **Push your code to GitHub first** (if not already):
   ```powershell
   cd C:\Users\bdene\CascadeProjects\windsurf-project-2
   git init
   git add .
   git commit -m "Initial commit"
   gh repo create nexus-ai --public --source=. --push
   ```
   *(Or create the repo on github.com manually and push.)*

3. In Render dashboard: **New → Blueprint** → connect your GitHub → pick the `nexus-ai` repo.

4. Render will read `backend/render.yaml` and pre-fill the service. Click **Apply**.

5. On the next screen, paste each secret you collected:
   - `GROQ_API_KEY` = your Groq key
   - `HF_API_TOKEN` = your HuggingFace token
   - `QDRANT_URL` = your Qdrant endpoint URL
   - `QDRANT_API_KEY` = your Qdrant API key
   - `DATABASE_URL` = your Neon connection string
   - *(`JWT_SECRET` is auto-generated by Render — no action needed)*

6. Click **Create resources**. First build takes ~5 min (Docker image build).

7. When the service shows **Live** with a green dot, copy the public URL — it looks like `https://nexus-ai-backend-xxxx.onrender.com`.

8. ☑ Save as **`API_BASE_URL`**

9. **Smoke-test it**:
   ```
   https://nexus-ai-backend-xxxx.onrender.com/health
   ```
   You should see:
   ```json
   {
     "status": "ok",
     "model": "llama-3.3-70b-versatile",
     "groq_configured": true,
     "embeddings_configured": true,
     "vector_db_configured": true,
     "auth_required": true
   }
   ```
   If any `_configured` is `false`, go to Render → Environment → fix that variable → save (it auto-redeploys).

---

## 6. Rebuild the Flutter APK pointing at production

```powershell
cd C:\Users\bdene\CascadeProjects\windsurf-project-2
flutter build apk --release --dart-define=API_BASE_URL=https://nexus-ai-backend-xxxx.onrender.com
```

APK will be at `build\app\outputs\flutter-apk\app-release.apk`. Install on your phone — works anywhere with internet, no laptop needed.

## Important caveats about the free tier

| Service | Free tier limit | Impact |
|---------|----------------|--------|
| **Render** | Service sleeps after 15 min of inactivity | First request after sleep takes ~30 s to wake up. Use `https://uptimerobot.com/` (free) to ping `/healthz` every 5 min if you want it always warm. |
| **Neon** | 0.5 GB Postgres | Plenty for tens of thousands of messages. |
| **Qdrant** | 1 GB vector storage | ~2 million chunks. Way more than you need. |
| **HuggingFace** | Generous free Inference API | Embedding ~50 docs/min. Watch out for `503` cold starts — the code already retries once. |
| **Groq** | 30 req/min, 14 400 req/day on free tier | More than enough for a demo. |

## Quick sanity command after deploy

```powershell
# Signup (creates an account)
$body = @{ email="test@example.com"; password="secret123"; name="Demo" } | ConvertTo-Json
$tok = (Invoke-RestMethod -Method POST -Uri "https://nexus-ai-backend-xxxx.onrender.com/auth/signup" -ContentType "application/json" -Body $body).access_token

# Test auth-protected endpoint
Invoke-RestMethod -Uri "https://nexus-ai-backend-xxxx.onrender.com/stats" -Headers @{ Authorization = "Bearer $tok" }
```

Expected: `{ documents: 0, chunks: 0, conversations: 0, messages: 0, model: 'llama-...' }`.
